Calibration & EEPROM Safety Guide

Quick operating reference and safety rules for the Intel macOS-focused OpenHantek6022 modification.

How to Use — Quick Reference

Use the routine path for ordinary offset calibration. EEPROM preparation is a separate, advanced procedure and is normally unnecessary.

Routine offset calibration

  1. Warm the oscilloscope under normal operating conditions, then short both analog inputs.
  2. Enable CH1 and CH2. Select a slow timebase that produces a 10–100 kS/s sample rate.
  3. Choose Oscilloscope → Calibrate Offset, then select Apply.
  4. Slowly visit all eight voltage ranges on both channels. Wait for each completion report before moving on.
  5. Turn calibration off only after all 16 channel/range combinations are reported complete.
  6. The verified INI calibration becomes active immediately. The EEPROM was not changed.
Normal stopping point: Once the 16 measurements complete and the INI is saved, routine calibration is finished.

EEPROM decision path

  1. Complete routine offset calibration first.
  2. Choose Oscilloscope → EEPROM Calibration Safety.
  3. Keep the null half-width at 0.30 ADC count unless new evidence supports another value.
  4. Leave the advanced EEPROM-update option unchecked, then run the read-only dry run.
  5. Review the report, proposed candidate, exact backup, and checksums in the timestamped safety folder.
  6. Stop if the result says NO MATERIAL CHANGE; EEPROM NOT WRITTEN.
Never select the advanced option merely to test it. It authorizes a physical EEPROM change when the reviewed candidate differs materially.

Normal Offset Calibration

The oscilloscope can have a noticeable zero-point error. Normal offset calibration measures the remaining error and stores device-specific residual corrections in an INI file. It does not write the oscilloscope EEPROM.

Before starting

  • Allow the oscilloscope and Mac to reach representative operating conditions.
  • Short both inputs with 50 Ω terminating plugs or properly shorted probe inputs.
  • Enable both channels.
  • Select a 10–100 ms/div timebase, or otherwise verify that the displayed sample rate is 10–100 kS/s.
  • Disconnect test signals. The application must be measuring the shorted zero level.

Collecting all 16 results

  1. Open Oscilloscope → Calibrate Offset.
  2. Read the preparation prompt. Use its Help button to return to this section if needed.
  3. Select Apply.
  4. Visit 20, 50, 100, 200, and 500 mV/div, then 1, 2, and 5 V/div on CH1 and CH2. A convenient approach is to place both channels on one range, wait for both completions, then advance both channels to the next range.
  5. Wait for the status bar to report each range complete. Do not sweep quickly through the selectors.
  6. Turn Calibrate Offset off only after all 16 channel/range combinations are complete.

The first frame after each range change is discarded. The application then requires two valid, stable measurements. Clipped, out-of-range, and unstable frames are rejected and retried.

An incomplete run is intentionally discarded. Turning calibration off before all 16 results complete cancels the run and leaves the active INI unchanged.

A completed calibration is saved and verified immediately. If an INI already exists, it is preserved as *.ini.bak. If replacement or verification fails, the previous file is restored.

Calibration Files and Sample-Rate Behavior

On macOS, device-specific calibration files are stored under:

~/Library/Application Support/OpenHantek/OpenHantek6022/Calibration

Choose Oscilloscope → Show Calibration Folder to reveal the active folder in Finder. When the native location is first used, an existing file under ~/.config/OpenHantek is copied and verified; the legacy original is retained as a migration backup.

Operating rate Hardware values INI residuals
Below 30 MS/s Low-speed EEPROM calibration [offset]
30 MS/s and above Protected high-speed EEPROM calibration [offset_high]

A normal low-speed calibration changes [offset] only.

EEPROM Calibration Safety

Oscilloscope → EEPROM Calibration Safety is an advanced, manual operation for evaluating whether saved low-speed INI residuals should be incorporated into hardware calibration. It is not part of routine offset calibration.

The default is read-only. Leave Also back up and update the device EEPROM (advanced) unchecked. A dry run creates the complete evidence bundle without sending an EEPROM write command.

What every dry run or update does first

  1. Reads the complete 80-byte calibration region twice.
  2. Requires the two reads to agree exactly.
  3. Creates a timestamped folder under Calibration/EEPROM Backups.
  4. Saves the exact EEPROM backup and an active INI snapshot.
  5. Builds a proposed low-speed candidate after applying the selected null rule.
  6. Writes a readable report and SHA-256 manifest for review.

The report records the raw and effective residual, null-window decision, calculated centre, and original and proposed bytes for every channel and voltage range.

Choosing the Null Window

The zero-centred null half-width controls only how a low-speed EEPROM candidate is prepared. It does not modify the active INI, change normal calibration measurements, or alter protected high-speed EEPROM values.

Selection Candidate rule Guidance
0.00 Null filtering is disabled; every finite residual is retained. Use only for deliberate analysis. It is more likely to retain measurement noise.
0.01–0.29 Ignore a residual when abs(residual) <= half-width; retain it when outside. More sensitive than the validated default and may propose unnecessary changes.
0.30 Recommended hardware-validated default for this fork; selected from eight runs and 128 results.
0.31–0.50 May suppress a real low-speed correction that the default would retain.

The repeatability dataset produced a statistical starting value of 0.29 ADC count. The production default was rounded upward to 0.30. The selector resets to 0.30 each time the dialog opens and is never persisted.

Keep 0.30 for routine use. Change it only when new repeatability data from representative hardware, temperature, and power-cycle conditions supports a different threshold. Validation on one unit is evidence for this fork's default, not a guarantee that every compatible oscilloscope has identical noise behavior.

After filtering and EEPROM-byte quantization, the complete proposed 80-byte image is compared with the current EEPROM. An identical image always produces NO MATERIAL CHANGE; EEPROM NOT WRITTEN, even if the advanced option was selected.

Rules for a Physical EEPROM Update

A physical update is persistent hardware maintenance. Do not proceed merely because the option exists. A recent, reviewed dry run must show a materially different candidate and justify the change.
  1. Complete and save a normal offset calibration.
  2. Run a fresh read-only dry run using the intended null half-width.
  3. Review the report, candidate bytes, exact original backup, and SHA-256 manifest.
  4. Stop if the candidate is identical or the evidence does not clearly justify a hardware change.
  5. Only for a reviewed, materially different candidate, reopen EEPROM Calibration Safety, reselect the null half-width, explicitly check the advanced option, and accept the separate physical-write confirmation.
  6. Keep oscilloscope power, USB, and Mac power stable until the verified result is displayed.
  7. Do not unplug the device, suspend the Mac, close the application, or disturb the USB link during the transaction.
  8. Preserve the timestamped safety folder after the result.

An authorized transaction can write only four aligned 8-byte low-speed offset chunks at EEPROM addresses 0x08, 0x10, 0x38, and 0x40. It then reads all 80 bytes and requires an exact match with the candidate.

After a verified update, low-speed [offset] residuals are zeroed to prevent double correction. Existing high-speed corrections remain under [offset_high], and [eeprom] replace_eeprom=false is verified.

Understanding the Result

Result Meaning Next action
No material change The proposed 80-byte image exactly matches the current EEPROM. No write occurred. Stop. Continue using the verified INI and existing EEPROM.
Read-only candidate differs The dry run found a proposed low-speed change. No write occurred. Review every report field and checksum. Proceed only if the change is justified.
Update verified The restricted chunks were written and the complete readback matched exactly. Preserve the bundle and confirm normal operation after a controlled power cycle.
Update stopped safely A problem was detected and the original chunks were restored and verified. Preserve the bundle. Do not retry until its report has been reviewed.
Recovery required The application could not verify a complete restoration after a failure. Do not attempt another write. Preserve all files and obtain a detailed review before recovery work.

Recovery and Retained Evidence

Any detected write, readback, INI, or audit failure triggers an attempt to restore the original low-speed chunks, followed by another complete readback. The transaction folder is a safety record, not disposable debug output.

Retain, as applicable:

  • the exact original EEPROM backup;
  • the proposed candidate and verified device readback;
  • the before-and-after INI snapshots;
  • the transaction report, state, and result records; and
  • the SHA-256 manifest.

Use Oscilloscope → Show Calibration Folder to find the calibration root. Each EEPROM operation stores its evidence under the device-specific EEPROM Backups directory.

Manual Command Is Separate

Oscilloscope → Manual command is retained for intentional low-level troubleshooting. It is not part of normal calibration, the EEPROM dry-run workflow, or routine recovery.

Do not use Manual Command to imitate EEPROM maintenance unless you understand the device protocol and have a specific reviewed recovery plan. The guarded EEPROM workflow provides backups, restricted writes, readback, rollback, and audit records that a raw command does not.
